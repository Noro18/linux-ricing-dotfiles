# Hyprland

The Wayland compositor managing windows, borders, shadows, and visual effects.

---

## File Structure

```
~/.config/hypr/
├── hyprland.conf             # Main Hyprland configuration
├── hyprland-gui.conf         # GUI-specific overrides (theme, colors, etc.)
├── colors/                   # Generated color configurations
│   └── colors.conf           # ⚠️ Generated by matugen on wallpaper change — do not edit
├── configurations/           # Modular configuration files (keybinds, monitors, etc.)
├── scripts/                  # Helper scripts (e.g., matu.sh for color generation)
├── hypridle.conf             # Idle timeout configuration
├── hyprlock.conf             # Lock screen configuration
├── hyprpaper.conf            # Wallpaper daemon configuration
├── hyprsunset.conf           # Sunset/sunrise color adjustment
├── plugins.conf              # Plugin loading configuration
├── env                       # Environment variables
├── monitors.conf             # Monitor layout configuration
├── workspaces.conf           # Workspace rules and names
└── windows-mode.conf         # Windows-specific behavior tweaks
```

---

## How the Files Relate

- `hyprland.conf` is the main configuration file that imports other modular configs.
- `hyprland-gui.conf` contains GUI-related settings (borders, shadows, rounding) and is where colors are applied.
- The `colors/` directory contains `colors.conf`, which is generated by `matugen` and defines all color variables used in the GUI.
- Other files in `configurations/` (like `keybinds.conf`, `monitors.conf`, etc.) are imported by `hyprland.conf` to keep the config organized.

---

## How Colors Work

Hyprland gets its colors from the `matugen` tool, which generates a Material You palette from the wallpaper.

| File | Generated by | When |
|------|-------------|------|
| `~/.config/hypr/colors/colors.conf` | matugen | On every wallpaper change |

The `colors.conf` file defines color variables (e.g., `$background`, `$primary`, `$on_primary`, etc.) that are used in `hyprland-gui.conf` to set:
- Window borders
- Inactive/active border colors
- Shadows
- Special rendering effects (like blur and glow)

To change how colors are applied to Hyprland elements, edit the `matugen` template:
- `~/.config/matugen/templates/hyprland-colors.conf`

This template maps Material You color tokens to the Hyprland color variables.

## Key Configuration Files

### hyprland-gui.conf
Controls the visual appearance of windows:
- Border size and color (active/inactive)
- Shadow size, color, and blur
- Corner rounding
- Background blur and opacity
- Animations (if enabled)

### colors.conf (generated)
Contains the color palette sourced by `hyprland-gui.conf`. Example usage in `hyprland-gui.conf`:
```conf
# Border colors
col.active_border: $blue
col.inactive_border: $background_75

# Shadow
shadow.range: 8
shadow.color: #00000080
```

### Other Modular Configs
- `keybinds.conf`: Keyboard shortcuts
- `monitors.conf`: Monitor layout and resolution
- `workspaces.conf`: Workspace names and rules
- `env`: Environment variables (like `$TERM`, `$EDITOR`)
- `hypridle.conf`: Idle actions (lock screen, dim, etc.)
- `hyprlock.conf`: Lock screen appearance
- `hyprpaper.conf`: Wallpaper settings (used by `swww` for setting wallpapers)
- `hyprsunset.conf`: Adjusts color temperature based on time of day

---

## ⚠️ Things Not to Edit Manually

- `~/.config/hypr/colors/colors.conf` — overwritten by matugen on wallpaper change
