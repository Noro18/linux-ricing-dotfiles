# SwayNC Notification Center

SwayNC (Superlative Wayland Notification Center) is the notification daemon and center used in this rice, providing a modern, themeable notification system that integrates seamlessly with the dynamic theming pipeline.

## Overview

SwayNC handles both:
- **Notification Center**: A panel-accessible history of notifications
- **Popup Notifications**: Real-time toast-style notifications

The component is fully integrated with the Matugen and Wallust theming systems, allowing colors to dynamically adapt to wallpaper changes.

## Configuration Files

### Main Configuration
- **Location**: `dotfiles/.config/swaync/config.json`
- **Purpose**: Controls behavior, positioning, widgets, and functionality

### Styling
- **Main Stylesheet**: `dotfiles/.config/swaync/style.css`
- **Color Files**: 
  - `dotfiles/.config/swaync/colors.css` (base colors)
  - `dotfiles/.config/swaync/colors-swaync.css` (SwayNC-specific colors)
- **Theme**: `dotfiles/.config/swaync/themes/matugen-nc/` (notification and control center themes)

## Features

### Dynamic Theming
Colors are automatically generated by Matugen and Wallust and applied through the CSS import system in `style.css`:
```css
@import 'colors-swaync.css';
@import 'colors.css';
@import 'themes/matugen-nc/notifications.css';
@import 'themes/matugen-nc/control-center.css';
```

### Control Center Widgets
The notification center includes various widgets:
- **Label**: Decorative separator
- **Buttons Grid**: Quick access to system controls (audio, Bluetooth, network, etc.)
- **Volume**: Audio volume control
- **Backlight**: Screen brightness control
- **MPRIS**: Media player integration
- **Title**: Section header
- **DND**: Do Not Disturb toggle
- **Notifications**: Notification list

### Keyboard Shortcuts
Enabled by default for quick access to notification actions.

## Integration

### Theme Synchronization
SwayNC colors are synchronized with the rest of the desktop through:
1. Wallpaper change triggers Matugen/Wal
2. Generated colors update `colors.css` and `colors-swaync.css`
3. SwayNC automatically reloads to apply new colors

### Launching
- **Notification Center**: Typically accessed via Super + N or system tray
- **Popups**: Appear automatically for new notifications

## Customization

### Modifying Appearance
Edit the theme files in `dotfiles/.config/swaync/themes/matugen-nc/`:
- `notifications.css` - popup notification styling
- `control-center.css` - notification panel styling

### Adjusting Behavior
Modify `dotfiles/.config/swaync/config.json` to change:
- Position (`positionX`, `positionY`)
- Sizes (`control-center-width`, `notification-window-width`)
- Timeouts (`timeout`, `timeout-low`, `timeout-critical`)
- Enabled widgets
- Widget configurations

## Troubleshooting

### Notifications Not Appearing
1. Verify swaync is running: `ps aux | grep swaync`
2. Check logs: `journalctl --user -u swaync`
3. Ensure configuration is valid JSON

### Theme Not Updating
1. Confirm Matugen and Wallust are running and generating colors
2. Check that `colors.css` and `colors-swaync.css` are being updated
3. Restart swaync to reload styles: `pkill swaync && swaync`

## Related Components

- **Waybar**: Status bar that may display notification indicators
- **Rofi**: Used for some system menus accessible from SwayNC
- **Matugen**: Color generation engine (part of dynamic theming)
- **Wallust**: Alternative color generation engine (part of dynamic theming)
