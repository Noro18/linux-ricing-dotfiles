# Waybar

The top bar showing workspaces, clock, system stats, and custom modules.

---

## File Structure

```
~/.config/waybar/
├── config.jsonc            # Module layout and config — the only file you edit for modules
├── style.css               # ⚠️ Generated by waybar-set — do not edit manually
├── colors.css              # ⚠️ Generated by matugen on wallpaper change — do not edit
├── colors-waybar.css       # ⚠️ Generated by wallust on wallpaper change — do not edit
├── custom styles/          # Style presets — one is active at a time via waybar-set
├── scripts/                # Scripts used by Waybar modules
└── generated/              # Other generated output files
```

---

## How the Files Relate

```
config.jsonc          ← edit this for module changes (single layout, never swapped)
style.css             ← generated by waybar-set, do not edit
  ├── @import colors.css           ← matugen colors (wallpaper-driven)
  ├── @import colors-waybar.css    ← wallust colors (wallpaper-driven)
  └── @import custom styles/X.css  ← whichever style is currently active
```

`style.css` is just a thin file that imports the three sources above. Its only job is to point at the active style preset — `waybar-set` rewrites it whenever you switch styles.

---

## Changing the Active Style

Run `waybar-set` (via keybind or `waybar-menu`) to pick a style from `custom styles/`. It rewrites `style.css` to import the selected preset and reloads Waybar.

To add a new style: drop a `.css` file into `custom styles/` and it will appear in the selector.

---

## Changing Modules / Layout

Edit `config.jsonc` directly — this file is never touched by `waybar-set` or the theming pipeline.

---

## How Colors Work

| File | Generated by | When |
|------|-------------|------|
| `colors.css` | matugen | On every wallpaper change |
| `colors-waybar.css` | wallust | On every wallpaper change |

To change how colors map to Waybar elements, edit the generator templates — not the output files:
- `~/.config/matugen/templates/` for Material You colors
- `~/.config/wallust/templates/` for terminal palette colors

---

## ⚠️ Things Not to Edit Manually

- `style.css` — rewritten by `waybar-set` every time you switch styles
- `colors.css` — overwritten by matugen on wallpaper change
- `colors-waybar.css` — overwritten by wallust on wallpaper change
